## The checkout

<table>
<colgroup>
<col style="width: 10%" />
<col style="width: 89%" />
</colgroup>
<thead>
<tr class="header">
<th>Feature</th>
<th>Details</th>
</tr>
</thead>
<tbody>
<tr>
<td>The basket and stored basket</td>
<td><div class="content-wrapper">
<p>The basket allows to change quantities and to set a remark per order line if this was enabled in the configuration settings. </p>
<p><img src="attachments/23561031/23571141.png" class="confluence-embedded-image" height="400" /></p>
<p>The basket can be stored as a new stored basket.</p>
<p><img src="attachments/23561031/23563173.png" class="confluence-embedded-image" height="400" /></p>
<p>In a stored basket the products can be edited, deleted or added to the active shopping cart.</p>
<p><img src="attachments/23561031/23571146.png" class="confluence-embedded-image" height="400" /></p>
</td>
</tr>
<tr>
<td>Checkout</td>
<td><div class="content-wrapper">
<p>The checkout is set up on one page and offers 5 steps.</p>
<p>eZ Commerce<br />
Step 1: A user can register as a new customer or he can buy as a guest.</p>
<p>Customers with an existing login can login and proceed to the checkout process</p>
<p><img src="attachments/23561031/23563156.png" class="confluence-embedded-image" height="400" /></p>
<p>Advanced version only</p>
<p>Step 1: A user can register as a new customer, existing customer or he can buy as a guest.</p>
<p>Customers with an existing login can login and proceed to the checkout process</p>
<p><img src="attachments/23561031/23568532.png" class="confluence-embedded-image" height="400" /></p>
</td>
</tr>
<tr>
<td>Invoice and delivery address</td>
<td><div class="content-wrapper">
<p>Step 2: Invoice address needs to be filled out by new customers or guests ordering and will be prefilled for logged in customers.<br />
<br />
</p>
<p><img src="attachments/23561031/23571139.png" class="confluence-embedded-image" width="650" /></p>
<p>Step 3: For delivery address the user can either choose the invoice address, one of the addresses from his address book or enter a new delivery address. For Advanced version only the offered delivery addresses will be fetched from the ERP.</p>
<p><img src="attachments/23561031/23567616.png" class="confluence-embedded-image" height="400" /></p>
<p><br />
</p>
</td>
</tr>
<tr>
<td>Delivery options</td>
<td><div class="content-wrapper">
<p>Step 4: The shipping methods and costs can be setup by a configuration.</p>
<p><img src="attachments/23561031/23563160.png" class="confluence-embedded-image" height="400" /></p>
<p><br />
</p>
<p><br />
</p>
</td>
</tr>
<tr>
<td>Payment options</td>
<td><p>The eCommerce solution offers a plugin for PayPal electronic payment (PayPal express). The payment is based on a standard Symfony Bundle. An eZ Partner can integrate other payment providers.</p></td>
</tr>
<tr>
<td>Order overview</td>
<td><div class="content-wrapper">
<p>Step 5: The last step summarizes the order. Checkboxes for accepting T&amp;C, Cancellation policy and data protection documents are offered.</p>
<p><img src="attachments/23561031/23570883.png" class="confluence-embedded-image" height="400" /></p>
</td>
</tr>
<tr>
<td>Order</td>
<td><p>The order is stored in the shop. A confirmation e-mail is sent to the customer and the email address which is set in the configuration.</p>
<p>Advanced version only</p>
<p>The order is forwared to the ERP system and also stored in the shop. A confirmation e-mail is sent to the customer and the email address which is set in the configuration.</p></td>
</tr>
</tbody>
</table>
