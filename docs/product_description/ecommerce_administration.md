#  eCommerce Administration 

eZ Commerce comes with a convenient configuration and management tools for the shop owner.

![](attachments/23561049/23568213.png)

<thead>
<tr class="header">
<th>Feature</th>
<th>Details</th>
</tr>
</thead>
<tbody>
<tr>
<td>Cockpit</td>
<td><div class="content-wrapper">
<p>The cockpit provides a good overview about the latest shop statistics, e.g. for sales or orders. It does not replace tools such as google analytics.</p>
<ul>
<li>Number of customers and active shopping carts</li>
<li>Top sellers</li>
<li>Top customers</li>
<li>Charts: orders and sales for the selected date(s)</li>
<li>Search statistics</li>
</ul>
<pre class="tw-data-text tw-ta tw-text-small"><code>Advanced version only</code></pre>
<ul>
<li>Status of the ERP system</li>
<li>Status of the server</li>
</ul>
<p><img src="attachments/23561049/23568154.png" class="confluence-embedded-image" height="400" /></p>
<p><br />
</p>
</td>
</tr>
<tr>
<td>Components</td>
<td><div class="content-wrapper">
<p>In the components section the shop admin will find the "Textmodules".</p>
<p><img src="attachments/23561049/23567969.png" class="confluence-embedded-image" height="150" /></p>
<p>The item silver.eShop gives an overview of the currently set configuration.</p>
<p><img src="attachments/23561049/23567852.png" class="confluence-embedded-image" height="400" /></p>
<p>See configurations can be changed in the configuration setting (see below).</p>
<p>In the item Textmodules all text concerning the shop can be managed as for basket, checkout, contact, customer center (if activated), email, footer, forms, order history and user.</p>
<p><img src="attachments/23561049/23567842.png" class="confluence-embedded-image" height="400" /></p>
<p><br />
</p>
</td>
</tr>
<tr>
<td>Order management</td>
<td><div class="content-wrapper">
<p>Orders are stored in the backend and can be filtered by date, status or payment information. An export function allows to export the data as XML file. </p>
<p><img src="attachments/23561049/23568053.png" class="confluence-embedded-image" height="400" /></p>
<p>If the shop is used without ERP integration invoices can be managed as well.</p>
</td>
</tr>
<tr>
<td>Configuration</td>
<td><div class="content-wrapper">
<p>The configuration settings in the backend allow to configure the main behaviour for the shop. In addition currency, payment and shipping options can be set per country (siteaccess). The configuration is stored using the version control features of the CMS.  More advanced settings (i.e. VAT for different countries) can be set up in configuration files.</p>
<p><img src="attachments/23561049/23568005.png" class="confluence-embedded-image" /></p>
</td>
</tr>
<tr>
<td>Price and shiping cost management</td>
<td><div class="content-wrapper">
<p>The Price and shiping costs management offers to setup prices and stock in case when the shop price provider (siso_price.price_provider.shop) is configured for calculating prices.</p>
<p><img src="attachments/23561049/23567799.png" class="confluence-embedded-image" height="250" /></p>
<p>See more details in the section <a href="23561048.html">"Manage products, prices and stock"</a>.</p>
</td>
</tr>

</table>
