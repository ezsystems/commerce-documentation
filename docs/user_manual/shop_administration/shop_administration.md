#  Shop administration 

eZ Commerce provides the following eCommerce functions in the top menu of the backend.

Depending on the chosen data provider and wether 3rd party software (ERP) has been integrated to the shop or not, there may be no data in some of the sub-tabs.

## Control Center

![](attachments/23560991/23565205.png)

<table>
<thead>
<tr class="header">
<th><p>Feature</p></th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>ERP request log</td>
<td>
<p><strong>Note:</strong> Only if an ERP system is connected!<br />
</p>
<p>All communication (request- and response messages), that is sent beetween shop and ERP system will be recorded and can bee seen here.</p>
<p><img src="attachments/23560991/23570835.png" class="confluence-embedded-image" height="400" /></p>
</td>
</tr>
<tr>
<td>ERP Performance</td>
<td>
<p><strong>Note:</strong> Only if an ERP system is connected!<br />
</p>
<p>The ERP performance viewer shows the efficiency of the connection between the shop and the ERP system in a monitoring.It allows to see the number of the different requests per date and time.</p>
<p>The monitoring tool regularly checks the connection to the ERP via a test request.</p>
<pre class="tw-data-text tw-ta tw-text-small"><code>If the connection has failed, the administrator receives an e-mail. If the connection is successful again, the alarm is ended by another e-mail.</code></pre>
<p><img src="attachments/23560991/23570837.png" class="confluence-embedded-image" height="400" /></p>
</td>
</tr>
<tr>
<td><p>eMail archive</p></td>
<td>
<pre class="tw-data-text tw-ta tw-text-small"><code>In the tab eMail archive the shop owner/admin finds all e-mails which were automatically sent from the shop, with date of despatch, recipient, subject and status. He can see the mails and resend them.</code></pre>
<p>The eMail archive reduces the response time for the support and allows to resend an eMail to the customer.</p>
<p><img src="attachments/23560991/23570799.png" class="confluence-embedded-image" width="600" /><br />
</p>
</td>
</tr>
<tr>
<td>eContent</td>
<td>
<p><strong>Note:</strong> Only if data provider is eContent!</p>
<p>If eContent is used as product data storage the shop owner can see all econtent types (product_group und product).</p>
<p>In the "eContent tasks" he can see how many products and product groups there are in the shop. Furthermore the shop operator can create a new index, import a backup into the database and create a backup from the database.</p>
<p><img src="attachments/23560991/23570833.png" class="confluence-embedded-image" height="400" /></p>
<p>In the tab "eContent types" he can see the econtent types with their associated attributes (product_group and product) and the mapping information.</p>
<p><img src="attachments/23560991/23570832.png" class="confluence-embedded-image" height="400" /></p>
</td>
</tr>
<tr>
<td>eCommerce jobs</td>
<td>
<p>The shop is using jobs to perform recurring tasks such as imports, backups etc. The job system lists the last jobs and shows the status and result of the job. </p>
<p><br />
</p>
<p><img src="attachments/23560991/23570801.png" class="confluence-embedded-image" width="600" /><br />
</p>
</td>
</tr>
</tbody>
</table>
